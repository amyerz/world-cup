# World Cup 2026 — Meta Portal app

A sideloadable Android app for **Meta Portal / Portal+** that wraps the World Cup 2026
React dashboard (from `design_handoff_worldcup_portal/`) in a full-screen WebView, with
all assets bundled locally (offline-capable) and an optional **live data** layer backed
by API-Football.

```
design_handoff_worldcup_portal/   # the original design handoff (source of truth — untouched)
web/                              # build pipeline (Node + esbuild)
  build.mjs        # JSX -> JS, vendor React/fonts, prefetch flags -> android assets/www/
  make-icon.mjs    # generates the launcher PNG
  src-extra/       # live-data.js (API adapter) + config.example.js
  config.js        # YOUR api key (gitignored) — created by you
android/                          # the Android app (Gradle, Java WebView host)
  app/src/main/java/com/erz/worldcup/MainActivity.java
  app/src/main/assets/www/        # GENERATED by web/build.mjs (do not edit by hand)
```

## Prerequisites (already set up on this machine)
- JDK 17, Android SDK (compileSdk 36), Node 20+
- `hzdb` (Horizon Debug Bridge): `npm i -g @meta-quest/hzdb`
- A Portal connected via USB-C with ADB enabled (Settings → Debug → ADB)

## Build & deploy
```bash
# 1. build the web bundle into android/app/src/main/assets/www/
cd web && node build.mjs

# 2. build the APK
cd ../android && JAVA_HOME="$(/usr/libexec/java_home -v 17)" ./gradlew assembleDebug

# 3. install + launch on the Portal
hzdb app install -r app/build/outputs/apk/debug/app-debug.apk
hzdb app launch com.erz.worldcup

# debug: console -> logcat tag "WCWeb"; screenshots:
hzdb adb logcat -d | grep WCWeb
hzdb capture screenshot -o /tmp/wc.png
```

## Data layer — free, no API key (default)
The app reads `window.WC_CONFIG.source`. The adapter (`web/src-extra/live-data.js`)
repopulates the app's existing `MATCHES` / `GROUPS` / `SCORERS` globals — no screen code
changes. Teams are matched by name (+ alias table). `PLAYERS` / stadiums / broadcasters
stay on bundled data (no free live source; largely static anyway).

| `source` | Cost | What it gives | Notes |
|---|---|---|---|
| **`openfootball`** (default) | **Free, no key** | Real 2026 schedule, groups, results (daily), **Golden Boot** from goal data; standings computed client-side | [openfootball/worldcup.json](https://github.com/openfootball/worldcup.json) via jsDelivr; bundled snapshot (`assets/www/data/worldcup-2026.json`) is the offline fallback. No in-match live minute (updates daily). |
| `apifootball` | Paid (for 2026) | Same + real-time minute + assists | api-sports.io free tier blocks season 2026; set `source:"apifootball"` + a paid `apiKey`. Budget-guarded at 95 req/day. |

Only the **second-by-second in-match ticker** and **player squads** aren't live on the free
path — everything else (schedule, groups, results, standings, top scorers, **bracket**) is real
and free. `refreshHours` (default 3) controls how often the openfootball JSON is re-pulled;
last-known data is cached in `localStorage` and the bundled snapshot covers full offline use.

### Live-only mode (current)
To avoid mixing live + mock data, non-live features are disabled:
- **Bracket** — now live from openfootball KO fixtures (TBD slots fill as teams advance).
- **Stats** — Top Scorers (live; "No goals yet" until the first goal) + **FIFA Ranking** (real,
  replaces the empty Assists panel). FIFA rank is also kept on team pages.
- **Hidden** (mock/heuristic/no-free-source): player squads, where-to-watch, chance-to-advance %,
  ceremony markers, coach.

Re-enable later by deleting the `Live-only` CSS block in `web/build.mjs` and reverting the
`screens.jsx` edits (Bracket/Stats/coach). The hides are CSS-only; the bracket/stats are source edits.

## Portal-specific notes
- `minSdk 28` (covers Portal + Portal+), `targetSdk 29`, no Google Mobile Services.
- Launcher needs `MAIN+LAUNCHER` + a PNG icon in `mipmap-xxxhdpi/` (both present).
- The ~64dp top system-overlay strip is handled in CSS: a `--portal-top-inset` band + a
  dark navy scrim (`.portal-topscrim`) keep the white system pills legible.
- Known framing artifact when pointed at 2022: the Home calendar is hard-coded to
  June/July 2026 and the Groups header says "12 groups" — both correct again on 2026 data.
